## DNS Infrastructure — Watchdog Deployment & Slack Alerting
**Why:** Deploy DNS drift watchdog as Azure Container App Job with Slack alerts
**What:** Built container job that runs every 15 minutes, compares live DNS records against Bicep IaC declarations, posts drift alerts to #system-alerts via Slack webhook. Deleted orphaned Function App resources from previous approach.
**Outcome:** Watchdog live and alerting. First real drift detected within 2 hours (stale CNAME for staging).
**On track?:** Focused
**Lessons:** Container App Jobs are better than Function Apps for scheduled infra checks — simpler lifecycle, no cold start issues, and the 15-minute schedule is more than enough for DNS monitoring.
